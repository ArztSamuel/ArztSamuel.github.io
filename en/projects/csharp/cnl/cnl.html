CNL - CampusNetLogin (V 1.0)

This small programm will automatically log into your account at the Campus internet at Puch Urstein.

Three step setup:

1
2
3
  1. Download the software for free and unzip the file.
  2. Run "CNLOptions.exe" and setup your login options.
  3. Put "CNL.exe" (or a shortcut) to it into your autostart-folder, or manually run it whenever you need a login.

Need help?

How does it work?

CNL will save all required data from "CNLOptions" encrypted on your local hard drive. CNL will then decrypt this data whenever you run it, in order to log you into your account. If you want to delete said save-data for whatever reason, you can push the button "Delete savedata" at the top right corner of CNLOptions.

Recommended "Advanced" Settings

The default settings are already the recommended settings. The next lines will explain options and different advantages of different settings.

Retry:
This option determines how often the program will retry to connect to the login page, if it fails the first time.
I would recommend setting this to either "5" or infinitely. However keep in mind that if you are not connected to the internet this will keep the program constantly running in the background, if set to "infinitely".

Check connection:
This option determines whether or not the program should check if you are still logged in every x-seconds.
I recommend turning this completely off. This is a comlicated one to explain but let me have a try...
I had to limit the cyclic repeat to minutes because checking your connection every second would sacrifice your bandwidth. Therefore setting this to "1" will check your connection every minute, which would mean in the worst case the program would automatically log you back in after 1 minute of being logged out. However you will most likely not be logged out if your connection is active every minute. So if you really want to turn this option on, I would recommend finding a happy medium between check time and worst case time... That is why the default value is "15": Worst case... 15 minutes. Most probable case... 7.5 minutes.

Open Debug Window:
Once you turn this option on a command prompt will always open when you run CNL.
This window solely serves debug purposes, therefore it is turned off by default. If you close the debug window you will also terminate the program. So if you turned "Check connection" on you will have to keep it at least minimized for it to work. If you want to see what exactly is happening while the program is running feel free to turn the debug window on. Or turn it on for... well... debug-reasons...
