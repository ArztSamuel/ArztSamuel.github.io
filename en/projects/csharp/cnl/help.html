Setup Guide

You can find a three step quick-setup guide here. If you still got troubles setting up CNL on your PC here is a more detailed step-by-step setup-guide:

  1. Download CNL.zip from here. (Save the "CNL.zip" on your computer)
  2. Unpack the zip file anywhere you like. (Right click -> Unpack)
  3. Run "CNLOptions.exe". (You might need to allow access to both .exe files)
  4. Configure your login. (Enter your username and password for the internet access, the rest is optional and can stay default)
  5. Apply the settings or simply close (automatically applies) the window.
  6. Now either put a shortcut (right click send to > desktop-shortcut) to "CNL.exe" into your windows autostart-folder or manually run it every time you want to be logged in.

In order to run the software you will only have to have Microsofts .Net-Framework installed on your computer (already installed on most machines).

Where is the windows autostart-folder?

The autostart folder varies from windows version:

Windows 7:
Go to "Start" > "Programmes" and search for the folder "autostart" and put a shortcut to "CNL.exe" in that folder.
Windows 8/10:
Either press "windows + R" and type in "shell:startup" to open your startup-folder or simply go to the directory "C:\Users\-Your User-\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup"

The Program does not start

If the program will not start when you try to run it, please verify that Microsoft's .Net Framework is installed on your machine.
If the program still won't startup please try deleting the save-folder "CNL". Simply type "%appdata%" into your directory field in explorer and delete the folder. (Directory: "C:\Users\-Your User-\AppData\Roaming")

If the save-folder does not exist and the program still won't start, please don't hesitate to contact me and I will do my best to resolve the issue.

I am not getting logged in

Please verify that the username and password you have entered in CNLOptions are correct.
If you are still not getting logged in try increasing the retry value in CNLOptions. If the problem persists please feel free to contact me under sarzt.mmt-b2015@fh-salzburg.ac.at
