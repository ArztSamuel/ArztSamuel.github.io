Setup Guide

You can find a three step quick-setup guide here. If you want to setup CNL on OSX you can find a guide here.
If you still got troubles setting up CNL on your PC, here is a more detailed step-by-step setup-guide for Windows:

  1. Download CNL.zip from here. (Save the "CNL.zip" on your computer)
  2. Unpack the zip file anywhere you like. (Right click -> Unpack)
  3. Run "CNLOptions.exe". (You probably need to allow execution of both .exe files)
  4. Configure your login. (Enter your username and password for the internet access, the rest is optional and can stay default)
  5. Apply the settings or simply close (automatically applies) the window.
  6. Now either put a shortcut (right click send to > desktop-shortcut) to "CNL.exe" into your windows autostart-folder or manually run it every time you want to be logged in.

In order to run the software you will only have to have Microsofts .Net-Framework installed on your computer (already installed on most machines).

In order to still allow the app to execute simply click on "Run" and tick off "Always ask before opening this file" on Windows 7 or on Windows 8/10 simply click on "More Info" and then click "Run anyway". Both processes are shown in the screenshots below (1 > Windows 7; 2, 3 > Windows 8/10).

Where is the windows autostart-folder?

The autostart folder varies from windows version:

Windows 7:
Go to "Start" > "Programmes" and search for the folder "autostart" and put a shortcut to "CNL.exe" in that folder.
Windows 8/10:
Either press "windows + R" and type in "shell:startup" to open your startup-folder or simply go to the directory "C:\Users\-Your User-\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup".

The Program does not start

If the program does not start when you try to run it, please verify that Microsoft's .Net Framework is installed on your machine.
If the program still won't startup please try deleting the save-folder "CNL". Simply type "%appdata%" into your directory field in explorer and delete the folder. (Directory: "C:\Users\-Your User-\AppData\Roaming")

If the save-folder does not exist and the program still won't start, please don't hesitate to contact me and I will do my best to resolve the issue.

I am not getting logged in

If you the problem only occurs on Autostart, please try to increase the value of "Retry" in CNLOptions (to 15 for example).
If you are also not getting logged in when manually executing CNL, please verify that the username and password you have entered in CNLOptions are correct.
If you are still not getting logged in after increasing the retry value and verifying your Logindata, please feel free to contact me at sarzt.mmt-b2015@fh-salzburg.ac.at.

Mac Support

Since this software was originally designed for Windows, setting it up on Mac requires a bit more effort. First of all you will need to install Mono (an independent version of Microsoft's .NET) on your machine. After installing Mono you are already able to run the .exe files by typing "<your path to Mono>/mono <your path to CNL>/CNLOptions.exe" into your terminal, where you insert your local path to where you installed Mono for <your path to Mono> and your local path to where you saved CNL for <your path to CNL>. So for example this would mean typing "/usr/bin/mono ~/downloads/CNL/CNLOptions.exe" into your terminal.
But this is very unwieldy. In order to launch the different .exe files by simply double clicking a file, you can also insert said paths into this script that you can download. Finally after that you will probably have to allow the script to be executed by typing "chmod +x <your path to runCNL script>/runCNL.sh". So in the end the content of the script could look like this:

"#!/bin/sh
exec /usr/bin/mono ~/downloads/CNL/CNLOptions.exe "$@""

This script will run CNLOptions. If you want to run CNL itself you can copy the script and simply change "CNLOptions.exe" to "CNL.exe". If you now put the script for running CNL.exe into your autostart-folder, CNL will be run when you startup your system.
If you changed the path of Shell on your mac you will also have to enter that path for "#!<your Shell path>/sh"
Please notice that when running CNLOptions for the first time on Mac it may take up to a minute for the window to appear. This is totally normal.

As I am not able to access a Mac for testing at the moment I can not assure any functionality, however all basic functions should work. If you yourself are a Mac-User and are able to test this program, please let me know if you encounter any problems at sarzt.mmt-b2015@fh-salzburg.ac.at.
